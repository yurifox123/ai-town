# AI生态小镇 - 生存系统设计文档

## 1. 概述

### 1.1 设计目标
- 增加游戏策略深度，Agent需要管理自身生存状态
- 促进Agent之间的互动（交易、帮助、竞争）
- 创造更真实的虚拟社会模拟

### 1.2 核心原则
- **可见性**：玩家能清楚看到Agent的生存状态
- **可预测**：属性变化有规律，Agent能做出合理决策
- **可恢复**：属性下降后有多种恢复途径
- **有挑战**：需要平衡工作和休息，不是无限资源

---

## 2. 属性系统

### 2.1 健康值 (Health)

| 项目 | 说明 |
|------|------|
| **范围** | 0 - 100 |
| **初始值** | 根据角色模板（80-100） |
| **作用** | 生命指标，影响行动能力 |

**变化规则：**
```
恢复：
- 睡眠：+10/小时
- 在家休息：+3/小时
- 公园锻炼：+5/次（消耗饱腹）

消耗：
- 饱腹=0时：-5/小时
- 健康<30时进入虚弱状态，行动速度减半
- 健康=0时昏迷，自动回家睡觉
```

**视觉效果：**
- 健康<50：角色头顶显示微弱呼吸动画
- 健康<30：角色 sprite 变暗
- 健康<10：角色闪烁红光

### 2.2 饱腹值 (Fullness)

| 项目 | 说明 |
|------|------|
| **范围** | 0 - 100 |
| **初始值** | 根据角色模板（70-90） |
| **作用** | 饥饿指标，影响健康和行动效率 |

**变化规则：**
```
自然消耗：
- 每小时：-3
- 移动时：额外-1/10格
- 工作时：额外-2/小时

饱腹状态效果：
- 80-100：饱腹，健康恢复速度+50%
- 40-79：正常
- 20-39：饥饿，健康缓慢下降，优先找食物
- 0-19：极度饥饿，健康快速下降，强制寻找食物
```

### 2.3 绿色积分 (Green Points)

| 项目 | 说明 |
|------|------|
| **范围** | -10000 ~ 10000000 |
| **初始值** | 10 |
| **作用** | 虚拟货币，用于交易和服务 |

**获取途径：**
```
工作收入：
- 咖啡馆工作：+20/小时
- 便利店工作：+15/小时
- 自由职业（写作/绘画）：+不定（随机消耗90~270小时，随机获得1000-10000积分，得到小积分的概率偏大）

其他：
- 完成任务：+50-200
- 社交互动：+5-20
- 系统奖励：+不定
```

**消费场景：**
```
食物：
- 咖啡：-5积分，+5饱腹
- 零食：-10积分，+10饱腹
- 便当：-30积分，+30饱腹
- 大餐：-50积分，+50饱腹

服务：
- 公园入场：-2积分
- 特殊活动：不定
```

## 4. AI决策影响

### 4.1 决策优先级

Agent的决策系统需要加入生存属性权重：

```
决策优先级（从高到低）：

紧急（立即处理）：
1. 健康=0 → 强制回家/找医生
2. 饱腹<20 → 立即寻找食物

高优先级：
3. 健康<30 → 停止工作，寻找休息地点
4. 饱腹<40 → 优先前往食物来源
5. 积分<0 → 寻找工作

正常：
6. 夜间（22:00-6:00）→ 倾向回家睡觉
7. 饱腹<70 → 考虑进食
8. 执行日常计划
```

### 4.2 行为模式变化

**饱腹值影响：**
- 80-100：积极社交，主动工作
- 40-79：正常行为
- 20-39：减少社交，专注找食物
- 0-19：拒绝社交，强制觅食

**健康值影响：**
- 70-100：全速移动，高效工作
- 40-69：移动速度正常
- 20-39：移动速度-50%，工作效率-30%
- 0-19：移动速度-80%，无法工作

## 5. 时间节奏

### 5.1 游戏时间比例

```
当前设置：
- 1秒现实时间 = 5分钟游戏时间
- 1小时现实时间 = 12小时游戏时间

饱腹消耗速度：
- 正常活动：约5-6小时游戏时间需要进食一次
- 现实时间约30分钟需要关注一次

健康恢复速度：
- 睡眠8小时（现实40分钟）可恢复满健康
```

### 5.2 昼夜循环

```
06:00 - 08:00：起床时间，Agent离开家
08:00 - 18:00：活动时间，工作/社交/购物
18:00 - 22:00：傍晚活动，聚餐/娱乐
22:00 - 06:00：睡眠时间，Agent回家休息

特殊：
- 饱腹<40时可能打破作息寻找食物
- 便利店24小时营业，咖啡馆夜间可能关闭
```

## 7. 技术实现方案

### 7.1 数据模型

```javascript
// Agent 属性（已在 agent.js 中）
this.health = {
  current: config.healthMax || 100,
  max: config.healthMax || 100
};
this.fullness = config.fullness || 80;
this.greenPoints = config.greenPoints || 10;

// 新增：上次更新时间（用于计算消耗）
this.lastUpdateTime = Date.now();
```

### 7.2 消耗计算

```javascript
// 在 tick 或时间更新时计算
updateSurvivalAttributes() {
  const now = Date.now();
  const elapsedMinutes = (now - this.lastUpdateTime) / 1000 * TIME_SCALE;
  
  // 饱腹自然消耗
  const fullnessConsumed = elapsedMinutes * 0.05; // 每小时3点
  this.fullness = Math.max(0, this.fullness - fullnessConsumed);
  
  // 健康根据饱腹状态变化
  if (this.fullness === 0) {
    const healthLost = elapsedMinutes * 0.083; // 每小时5点
    this.health.current = Math.max(0, this.health.current - healthLost);
  }
  
  this.lastUpdateTime = now;
}
```

### 7.3 AI提示词更新

## 9. 平衡性考虑

### 9.1 避免的问题

1. **太简单**：属性下降太慢，玩家无需关注
   - 解决：调整消耗速度，确保30分钟内需要干预

2. **太困难**：属性下降太快，Agent频繁死亡
   - 解决：提供多种恢复途径，降低死亡惩罚

3. **太重复**：Agent只关注生存，忽略社交/目标
   - 解决：生存和日常计划并行，不是互斥

### 9.2 测试指标

- Agent平均存活时间 > 2小时现实时间
- 每天需要进食 2-3 次
- 每天需要睡眠 1 次
- 积分收入 > 基本消耗
